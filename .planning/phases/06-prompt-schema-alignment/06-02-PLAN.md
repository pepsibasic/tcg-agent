  <behavior>
    **card_analysis contract:**
    - Test: card_analysis prompt mentions every schema key that the LLM should produce (identity_tags, rarity_signal, liquidity_signal, price_band, reasoning_bullets, confidence)
    - Test: card_analysis prompt does NOT mention card_id (orchestrator-injected)
    - Test: a sample object matching the prompt's field descriptions validates against CardAnalysisSchema

    **portfolio_summary contract:**
    - Test: portfolio_summary prompt mentions every LLM-produced key (concentrationScore, liquidityScore, collectorArchetype, missingSetGoals, recommendedActions)
    - Test: portfolio_summary prompt does NOT mention orchestrator-injected keys as fields to produce (userId, totalValueEst, breakdown, priceDataAsOf, priceConfidence)
    - Test: a sample object with all PortfolioSummarySchema keys validates against the schema

    **archetype_identity contract:**
    - Test: archetype_identity prompt mentions every schema key (name, traits, why, comparable_collectors, share_card_text, share_card_badges)
    - Test: archetype_identity prompt does NOT mention "badges" without the "share_card_" prefix in its field list
    - Test: a sample object matching the prompt's field descriptions validates against CollectorArchetypeSchema
  </behavior>
  <implementation>
    Create `packages/agent/src/llm/__tests__/prompt-schema-contract.test.ts`:

    1. Import PROMPTS from prompts.ts and all 3 schemas from @tcg/schemas
    2. For each prompt-schema pair, extract the LLM-produced field names that the prompt instructs (the ones listed after "Produce a JSON object")
    3. Verify the prompt mentions each expected schema key that the LLM is responsible for
    4. Verify the prompt does NOT instruct the LLM to produce orchestrator-injected fields
    5. Create a valid sample object per schema and verify it passes .safeParse()

    **Approach for field detection:** Use string matching against the rendered prompt user text. Check that the prompt contains each field name (e.g., `expect(user).toContain('concentrationScore')`). This is more resilient than regex extraction and catches both additions and removals.

    **Orchestrator-injected fields to exclude from prompt checks:**
    - card_analysis: card_id (orchestrator injects)
    - portfolio_summary: userId, totalValueEst, breakdown, priceDataAsOf, priceConfidence (orchestrator computes from DB)
    - archetype_identity: none (all fields come from LLM, badges overridden post-call)
  </implementation>
