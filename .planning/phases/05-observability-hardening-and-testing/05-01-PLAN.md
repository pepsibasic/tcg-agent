  <files>packages/agent/src/llm/generate.ts, packages/agent/src/llm/types.ts</files>
  <action>
    1. Extend GenerateWithRetryOptions to accept an optional `logger` parameter (typed as `{ info: (obj: Record<string, unknown>, msg: string) => void; warn: (obj: Record<string, unknown>, msg: string) => void }`) and optional `cardId` (string). This keeps the agent package decoupled from Fastify -- callers pass their request.log in.

    2. In the generateWithRetry for-loop, after each generateStructured call:
       - On success: log at info level with `logger.info({ card_id: cardId, model: config.model, attempts: attempt, latency_ms: elapsed }, 'llm_generation_success')`. Measure latency with `Date.now()` before/after the generateStructured call. If the Vercel AI SDK result includes token usage (check result object for `usage` property with `promptTokens` and `completionTokens`), include `input_tokens` and `output_tokens` in the log object.
       - On validation failure (result.success === false): log at warn level with `logger.warn({ card_id: cardId, model: config.model, attempt, error_path: result.failure.reason.slice(0, 500), raw_output_truncated: JSON.stringify(result.failure.partial).slice(0, 500) }, 'llm_validation_failure')`. This satisfies OBS-03.

    3. After the retry loop exhausts (final failure return), log at warn level: `logger.warn({ card_id: cardId, model: config.model, total_attempts: maxAttempts, final_error: lastError.slice(0, 500) }, 'llm_generation_exhausted')`.

    4. If logger is not provided (backward compatibility), skip all logging -- use `if (options.logger)` guards. This ensures existing tests and callers work without changes.

    5. Update the orchestrator callers in packages/agent (card analysis, portfolio summary, archetype orchestrators) to pass through logger and cardId when available. If orchestrator functions don't currently accept logger, add optional logger parameter to their options types.
