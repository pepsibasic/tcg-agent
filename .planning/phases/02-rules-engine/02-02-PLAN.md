<objective>
Implement the per-card action eligibility rules for all 4 card states using TDD. Each state handler is a pure function that maps (card state + price confidence + listing status + pack context) to an Action[] array.

Purpose: This is the core of RULE-01 (deterministic action eligibility), RULE-02 (all action types with params/ui_copy/risk_notes), and RULE-03 (rules engine is sole source of actions field). The VAULTED state has the most complex conditional logic (BUYBACK gated on price data, LIST suppressed when already listed, risk_notes on stale prices). Tests drive the implementation to ensure every edge case is covered.

Output: 4 state action builders, 4 test files, updated dispatcher with real implementations replacing placeholders.
</objective>
