<objective>
Implement the vault conversion recommendation engine using TDD. This covers cross-card analysis: identifying which external cards should be vaulted (single-card threshold) and when batch shipping makes sense (card count or total value threshold).

Purpose: VAULT-01 (configurable threshold triggers), VAULT-02 (unlocks reasons on every recommendation), VAULT-03 (batch shipping prompt). The vault conversion function operates on arrays of external cards — a portfolio-level analysis distinct from the per-card computeEligibleActions in Plan 02. It produces SHIP_TO_VAULT actions for individual qualifying cards and a single BUNDLE_SHIP action when batch conditions are met.

Output: `vault/conversion.ts` with full implementation, `vault/unlock-reasons.ts` for dynamic unlock text, test file with threshold combination coverage.
</objective>

       if value >= config.vaultSingleCardThreshold:
         push SHIP_TO_VAULT action with:
           params: { cardValue: value, unlocks: buildUnlockReasons(card), batchEligible: isBatchEligible } as Record<string, unknown>
           ui_copy: "Ship to Vault — Unlock instant liquidity and verified ranking for this $${Math.round(value)} ${card.cardName}"

     ```
     Define `function computeIdentityVaultTrigger(_cards: ExternalCardInput[]): boolean { return false }` as a module-private function. This is the Phase 4 activation point.

     **Batch BUNDLE_SHIP:**
     ```
     const totalValue = sum of all card estimatedValues (null → 0)
     const isBatchEligible = externalCards.length >= config.batchCardCountThreshold || totalValue >= config.batchTotalValueThreshold

     if isBatchEligible:
       const savings = (externalCards.length - 1) * config.bundleShipSavingsPerCard
       push BUNDLE_SHIP action with:
         params: { cardCount: externalCards.length, totalValue, estimatedSavings: savings } as Record<string, unknown>
         ui_copy: "Bundle Ship to Vault — Ship ${externalCards.length} cards together and save ~$${savings}"
         risk_notes: null
