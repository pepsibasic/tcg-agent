    **packages/db/prisma/schema.prisma:**
    Create the complete data model exactly as specified in RESEARCH.md Pattern 4, which reflects all locked decisions from CONTEXT.md. The schema includes:

    Generator: prisma-client-js, output "../src/generated/client"

    Datasource: provider "postgresql", url env("DATABASE_URL") — note: Prisma 7 reads the actual URL from prisma.config.ts but the datasource block is still syntactically required.

    Enums:
    - CardState: VAULTED, EXTERNAL, ON_MARKET, IN_TRANSIT
    - PriceConfidence: LIVE, RECENT_24H, STALE_7D, NO_DATA

    Models (all PKs use @default(uuid(7)) @db.Uuid, all FKs use @db.Uuid):
    - User: id, email (unique), createdAt, updatedAt, deletedAt? (soft delete). Relations: userCards, externalCards, actionsLog.
    - Card: id, name, ipCategory (string — flexible enum per decision), setName?, language (ISO 639-1), grade? (string), imageUrl?, createdAt, updatedAt. Relations: userCards, packCards.
    - UserCard: id, userId, cardId, state (CardState), estimatedValue? (Decimal(10,2)), priceFetchedAt?, priceConfidence (PriceConfidence, default NO_DATA), certNumber?, userNotes?, createdAt, updatedAt, deletedAt?. Relations: user, card, marketplaceListing?. Indexes: [userId], [state].
    - ExternalCard: id, userId, name, setName?, grade?, certNumber?, estimatedValue? (Decimal(10,2)), priceConfidence (PriceConfidence, default NO_DATA), priceFetchedAt?, userNotes?, createdAt, updatedAt, deletedAt?. Relations: user. Indexes: [userId].
    - Pack: id, userId, name, ipCategory, openedAt?, createdAt, updatedAt. Relations: packCards.
    - PackCard: id, packId, cardId, createdAt. Relations: pack, card.
    - MarketplaceListing: id, userCardId (unique), listPrice (Decimal(10,2)), status (String, default "ACTIVE"), listedAt (default now()), updatedAt. Relations: userCard.
    - ActionsLog: id, userId, cardId?, agentRecommended (Json), userAction?, createdAt. Relations: user. Indexes: [userId].

    CRITICAL: Every @id field with uuid(7) MUST also have @db.Uuid. Every FK reference to a UUID field MUST also have @db.Uuid. Without this, Postgres uses VARCHAR(36) instead of native UUID type (Pitfall #6).

    **packages/db/prisma.config.ts:**
    Per Prisma 7 Pattern 2 from RESEARCH.md:
    ```typescript
    import 'dotenv/config'
    import { defineConfig, env } from 'prisma/config'

    export default defineConfig({
      schema: './prisma/schema.prisma',
      migrations: {
        path: './prisma/migrations',
      },
      datasource: {
        url: env('DATABASE_URL'),
      },
    })
    ```
    CRITICAL: Do NOT put adapter config here. The adapter belongs in client.ts, NOT in defineConfig (Pitfall #1 — adapter was removed from defineConfig in Prisma 7 GA).

    **packages/db/src/client.ts:**
    Per Prisma 7 singleton pattern from RESEARCH.md:
    ```typescript
    import 'dotenv/config'
    import { PrismaClient } from '@prisma/client'
    import { PrismaPg } from '@prisma/adapter-pg'

    const adapter = new PrismaPg({
      connectionString: process.env.DATABASE_URL!,
    })

    const globalForPrisma = globalThis as unknown as { prisma?: PrismaClient }

    export const prisma =
      globalForPrisma.prisma ??
      new PrismaClient({
        adapter,
        log: process.env.NODE_ENV === 'development' ? ['query', 'error'] : ['error'],
      })

    if (process.env.NODE_ENV !== 'production') {
      globalForPrisma.prisma = prisma
    }
    ```
