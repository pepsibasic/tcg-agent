**Define types:**
```typescript
type ArchetypeSuccess = { success: true; data: ArchetypeResponse; degraded?: boolean }
type ArchetypeProgressNudge = { success: true; data: { archetype: null; progress: string; message: string } }
type ArchetypeResult = ArchetypeSuccess | ArchetypeProgressNudge
```

**Implement `computeArchetypeBadges(stats)`:**
```typescript
function computeArchetypeBadges(stats: {
  vaultedCount: number; topIpPercent: number; externalCount: number
}): string[] {
  const badges: string[] = []
  if (stats.vaultedCount >= 10) badges.push('vault_builder')
  if (stats.topIpPercent >= 0.6) badges.push('ip_specialist')
  if (stats.externalCount >= 5) badges.push('external_collector')
  return badges
}
```

**Pipeline:**
1. **Fetch data:** `prisma.userCard.findMany({ where: { userId, deletedAt: null }, include: { card: true } })` and `prisma.externalCard.findMany({ where: { userId, deletedAt: null } })`. Also fetch recent actions: `prisma.actionsLog.findMany({ where: { userId }, orderBy: { createdAt: 'desc' }, take: 20 })`.

2. **Check threshold:** `const totalCards = userCards.length + externalCards.length`. If `totalCards < 5`, return progress nudge: `{ success: true, data: { archetype: null, progress: '${totalCards}/5 cards', message: 'Add ${5 - totalCards} more cards to unlock your collector identity!' } }`.

3. **Compute portfolio stats for badges:**
   - `vaultedCount`: userCards where state === 'VAULTED'.length
   - IP distribution: count cards per ipCategory (userCards via card.ipCategory; externalCards count as their own category or use 'External'). Find top IP percentage: `topIpPercent = maxCount / totalCards`.
   - `externalCount`: externalCards.length

4. **Compute deterministic badges:** `const badges = computeArchetypeBadges({ vaultedCount, topIpPercent, externalCount })`.

5. **Compute top IPs string:** Sort ipCategories by count descending, take top 3, format as comma-separated string.

6. **Render prompt:** `renderPrompt('archetype_identity', { user_id: userId, portfolio_summary_json: JSON.stringify({ totalCards, vaultedCount, externalCount: externalCards.length, topIps: ipDistribution }), top_ips: topIpsString, action_history: JSON.stringify(recentActions.map(a => a.userAction).filter(Boolean)) })`.

7. **Call LLM:** `generateWithRetry({ schema: CollectorArchetypeSchema, prompt, system, config: DEFAULT_LLM_CONFIG, narrativeFields: ['why', 'share_card_text', 'traits', 'comparable_collectors'] })`.

8. **Success path:** Override the LLM's `share_card_badges` with deterministic badges: `{ success: true, data: { ...llmResult.data, share_card_badges: badges } }`. This prevents LLM hallucinating badge names that don't exist in the UI (per Research pitfall #5).

9. **Degraded path:** Return defaults with deterministic badges still present: `{ success: true, degraded: true, data: { name: 'Collector', traits: [], why: '', comparable_collectors: [], share_card_text: '', share_card_badges: badges } }`.

**Update barrel export:** Add `export { detectArchetype } from './archetype.js'` to `packages/agent/src/orchestrators/index.ts`.

**Key decisions:**
- Badges are computed AFTER the LLM call and OVERRIDE the LLM output — deterministic, testable, predictable per CONTEXT.md decision
- The prompt template field `badges` maps to schema field `share_card_badges` — the LLM will produce `share_card_badges` because the schema enforces the field name, but we override anyway to ensure determinism
