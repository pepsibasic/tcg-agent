# Milestone v1.0 Audit Report

**Audited:** 2026-03-05
**Status:** gaps_found
**Phases:** 5/5 verified (all passed per-phase verification)
**Requirements:** 28/43 fully satisfied, 15 partial (cross-phase wiring issues)

## Executive Summary

All 5 phases pass their individual phase verifications (code exists, tests pass, within-phase wiring correct). However, the cross-phase integration check reveals that **all 3 LLM prompt templates have field name/type mismatches with their target Zod schemas**, causing every LLM call to fail validation and fall to the degraded path. Additionally, the vault conversion engine is orphaned from the API surface.

The system "works" in a degraded mode — routes return 200 with fallback data — but the LLM intelligence layer never successfully produces validated output for any of the 3 agent orchestrators.

## Critical Findings

### 1. All 3 Prompt Templates Misalign with Zod Schemas

| Template | Prompt Says | Schema Expects | Impact |
|----------|------------|----------------|--------|
| `card_analysis` | `confidence: number (0.0-1.0)` | `confidence: z.enum(['HIGH','MEDIUM','LOW'])` | Always fails |
| `card_analysis` | `price_band: {low, mid, high}` | `price_band: {low, high, currency}` | Always fails |
| `portfolio_summary` | `concentration_score` (snake_case) | `concentrationScore` (camelCase) | Always fails |
| `portfolio_summary` | `breakdown: {vaulted, external, other}` | `breakdown: [{ipCategory, totalValue, cardCount, ...}]` | Always fails |
| `archetype_identity` | `badges: string[]` | `share_card_badges: z.array(z.string())` | Always fails |

**Root cause:** Prompt templates were written in Phase 3 before orchestrators were built in Phase 4. The prompts describe ideal field names but the Zod schemas (Phase 1) use different conventions.

**Why tests don't catch this:** All tests mock at the orchestrator boundary (`vi.mock('@tcg/agent')`) or mock `generateWithRetry` directly. No test exercises the real prompt → LLM → Zod validation pipeline.

### 2. Vault Conversion Engine Orphaned from API

`computeVaultConversionCandidates` has 30 passing unit tests but is never called from any route or orchestrator. Requirements VAULT-01/02/03 are implemented internally but not surfaced to users.

### 3. LLM Logger Never Propagated

`generateWithRetry` accepts an optional `logger` parameter for LLM diagnostics (OBS-03). No orchestrator passes one — `request.log` from Fastify is never threaded through. All LLM diagnostic events are silently dropped.

### 4. Batch Analyze Missing Audit Log

`POST /agent/card/analyze` (single) writes `RECOMMENDATION` to `actionsLog`. `POST /agent/card/analyze-batch` (pack pull) does not. OBS-02 partially satisfied.
